# 配置管理系统使用说明

## 概述

配置管理系统是一个独立的模块，用于管理系统的所有配置参数。它提供了完整的配置管理功能，包括本地配置文件管理、远程配置更新、配置导入导出等。

## 功能特性

### 🔧 核心功能
- **本地配置管理**: 管理 `config/config.ini` 文件
- **配置分类管理**: 按功能模块分类管理配置项
- **实时配置编辑**: 支持多种数据类型的配置项编辑
- **配置导入导出**: 支持 INI 格式的配置导入导出
- **远程配置更新**: 支持从远程服务器更新配置

### 🎨 界面特性
- **树形配置导航**: 左侧树形结构，快速定位配置项
- **分类标签页**: 右侧按配置节分类的编辑界面
- **智能控件**: 根据配置值类型自动选择合适的编辑控件
- **实时保存**: 配置修改后自动保存到文件

### 🌐 远程配置
- **HTTP API 支持**: 通过 HTTP GET 请求获取远程配置
- **JSON 格式**: 支持标准 JSON 格式的配置数据
- **异步更新**: 后台线程执行，不阻塞主界面
- **错误处理**: 完善的错误处理和用户提示

## 使用方法

### 方法1: 在主应用程序中使用

在主应用程序中，配置管理功能已集成到菜单栏中：

1. 启动主应用程序 (`core_ui_2.py`)
2. 点击菜单栏 `文件` → `⚙️ 配置管理` (快捷键: `Ctrl+Shift+C`)
3. 配置管理对话框将打开

### 方法2: 独立启动配置管理器

#### Windows 批处理文件
```bash
# 双击运行
launch_config_manager.bat
```

#### PowerShell 脚本
```powershell
# 在PowerShell中运行
.\launch_config_manager.ps1
```

#### Python 脚本
```bash
# 直接运行Python脚本
python launch_config_manager.py
```

## 配置项说明

### 数据库配置
- **MySQLConfig**: MySQL数据库连接配置
- **RedisConfig**: Redis缓存配置
- **InfluxDBConfig**: InfluxDB时序数据库配置

### 系统配置
- **paths**: 系统路径配置
- **Logging**: 日志系统配置
- **UISettings**: 用户界面配置

### 业务配置
- **DrivingThresholds**: 驾驶行为阈值配置
- **MQTTConfig**: MQTT消息队列配置
- **BehaviorConfig**: 行为分析算法配置

## 配置编辑

### 数据类型支持
- **字符串**: 使用 `QLineEdit` 控件
- **整数**: 使用 `QSpinBox` 控件
- **浮点数**: 使用 `QDoubleSpinBox` 控件
- **布尔值**: 使用 `QCheckBox` 控件

### 编辑操作
1. 在左侧树形结构中点击配置项
2. 右侧标签页会自动切换到对应的配置节
3. 修改配置值，系统会自动保存
4. 使用工具栏按钮进行批量操作

## 工具栏功能

### 💾 保存配置
- 手动保存当前配置到文件
- 快捷键: 无

### 📂 加载配置
- 从文件重新加载配置
- 会刷新界面显示

### 📤 导出配置
- 将当前配置导出为 INI 文件
- 支持选择保存路径

### 📥 导入配置
- 从 INI 文件导入配置
- 会覆盖当前配置

### 🌐 远程更新
- 从远程服务器更新配置
- 需要先设置远程配置URL

### 🔄 刷新
- 刷新配置显示
- 重新加载配置文件

## 远程配置更新

### 设置远程URL
1. 点击 `🌐 远程更新` 按钮
2. 输入远程配置URL (例如: `http://example.com/config.json`)
3. 确认更新操作

### 远程配置格式
远程配置需要返回标准的 JSON 格式：

```json
{
  "MySQLConfig": {
    "host": "remote-server.com",
    "port": "3306",
    "user": "remote_user"
  },
  "UISettings": {
    "theme": "dark",
    "language": "en_US"
  }
}
```

### 安全注意事项
- 确保远程服务器使用 HTTPS
- 验证远程配置的完整性
- 建议在更新前备份本地配置

## 故障排除

### 常见问题

#### 1. 配置管理器无法启动
- 检查 Python 是否安装 (版本 3.7+)
- 检查 PySide6 是否安装: `pip install PySide6`
- 检查项目路径是否正确

#### 2. 配置文件无法加载
- 检查 `config/config.ini` 文件是否存在
- 检查文件权限
- 检查文件编码 (应为 UTF-8)

#### 3. 远程配置更新失败
- 检查网络连接
- 验证远程URL是否正确
- 检查远程服务器是否可访问
- 验证远程配置格式是否正确

#### 4. 配置修改后未保存
- 检查文件写入权限
- 检查磁盘空间
- 查看错误日志

### 日志查看
配置管理器的操作日志会输出到控制台，包括：
- 配置文件加载/保存状态
- 远程更新进度
- 错误信息和异常堆栈

## 扩展开发

### 添加新的配置节
1. 在 `ConfigManager.create_default_config()` 方法中添加新节
2. 配置项会自动出现在界面中
3. 支持所有标准的数据类型

### 自定义配置验证
1. 继承 `ConfigManager` 类
2. 重写 `set_value()` 方法
3. 添加自定义验证逻辑

### 集成到其他模块
1. 导入 `ConfigManager` 类
2. 创建配置管理器实例
3. 使用 `get_value()` 和 `set_value()` 方法

## 技术架构

### 核心组件
- **ConfigManager**: 配置管理核心类
- **ConfigurationDialog**: 配置管理界面
- **RemoteConfigUpdater**: 远程配置更新线程

### 设计模式
- **观察者模式**: 使用信号槽机制通知配置变更
- **策略模式**: 支持多种配置存储方式
- **工厂模式**: 根据数据类型创建相应的编辑控件

### 线程安全
- 配置读写操作在主线程执行
- 远程更新在后台线程执行
- 使用信号槽机制进行线程间通信
