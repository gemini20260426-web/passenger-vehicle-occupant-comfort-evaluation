# 📁 多源异构数据同步系统 - 项目说明

## 🎯 项目概述

本项目是一个高性能的实时多源异构数据同步和融合系统，支持100Hz同步频率，毫秒级延迟，适用于工业物联网、传感器网络、实时监控等场景。

## 🔧 如何避免文件冲突

### 1. 文件名冲突已解决
- ✅ `main.py` → `multi_source_sync_system/main.py`
- ✅ `config_manager.py` → `multi_source_sync_system/core/multi_source_sync/config_manager.py`
- ✅ 所有相关文件都已移动到独立目录

### 2. 导入路径已调整
- 新系统使用相对导入，不会影响现有系统
- 每个系统都有独立的Python路径

### 3. 配置文件分离
- 现有系统：`config/` 目录
- 新系统：`multi_source_sync_system/config/` 目录

## 🚀 快速开始

### 进入项目目录
```bash

```

### 安装依赖
```bash

```

### 启动系统

#### 命令行模式


## 📋 系统特性

- **🔗 多源数据同步**: 支持MQTT、串口、文件、数据库等多种数据源
- **⏰ 智能时间对齐**: NTP同步、自适应策略、毫秒级精度
- **🔄 高级数据融合**: 加权平均、卡尔曼滤波、神经网络等算法
- **📊 实时性能监控**: CPU、内存、延迟、吞吐量等指标
- **🎯 数据质量评估**: 完整性、准确性、一致性检查
- **⚡ 异步并发处理**: 多线程优化、队列管理、自适应工作线程

## 🔍 与现有系统的区别

| 特性 | 现有UI系统 | 多源异构数据同步系统 |
|------|------------|----------------------|
| 主要功能 | 用户界面控制 | 数据同步和融合 |
| 运行方式 | GUI界面 | 命令行/GUI可选 |

## 🛠️ 开发建议

### 1. 独立开发
- 两个系统可以独立开发和维护
- 不会相互影响

### 2. 共享资源
- 如果需要共享某些功能，可以创建公共模块
- 建议放在 `utils/` 或 `common/` 目录

### 3. 测试隔离
- 每个系统都有独立的测试目录
- 可以独立运行测试
