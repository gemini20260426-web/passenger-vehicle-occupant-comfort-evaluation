### 2.6 [5] 波形滚动 — `WaveformScrollWidget` 🐛 **信号名 Bug**

| 检查项 | 结果 |
|--------|------|
| `set_data_bridge()` | 存在 (L329) |
| `update_frame_result()` | ❌ 不存在 |
| 数据链路 | **BUG: 连接到不存在的信号 `frame_ready`，应为 `frame_result_ready`** |
| 是否接收实时数据 | 🐛 **否 — 信号连接失败被 `try/except` 静默吞没** |

**Bug 详情**:

```python
# waveform_scroll_widget.py L329-338 — 当前代码:
def set_data_bridge(self, data_bridge):
    self._data_bridge = data_bridge
    if data_bridge:
        try:
            data_bridge.frame_ready.connect(self._on_frame_ready)       # ← 信号名错误!
            data_bridge.behavior_event_ready.connect(self._on_event_detected)  # ← 这个正确
        except Exception as e:
            self.logger.warning(f"连接 DataBridge 失败: {e}")           # ← 静默吞没
```

**DataBridge 实际信号**:
```python
# data_bridge.py L28:
frame_result_ready = Signal(object)    # ← 正确名称
# 不存在名为 frame_ready 的信号! ❌
```

**影响**: 波形滚动组件从未接收到任何实时加速度数据。`_on_frame_ready()` 和 `_on_event_detected()` 永远不会被调用。但 `try/except` 静默吞没异常，**不报错、不警告**，开发者无法从日志发现此 Bug。

**严重度**: 🔴 **高 — 数据链路中断，模块形同虚设**

def set_data_bridge(self, data_bridge):

## 六、结论

| 指标 | 结果 |
|------|------|
| 数据链路完整连通 | 3/8 (37.5%) — dashboard, timeline, feature |
| 间接连通 | 1/8 (12.5%) — ComparisonTab |
| 连通但 Bug 中断 | 1/8 (12.5%) — WaveformScroll (信号名错误) |
| 完全孤立 | 2/8 (25.0%) — ABModelComparison, EventReplayWidget |
| 独立运行 (不需实时数据) | 1/8 (12.5%) — MLTrainingPanel |

**核心问题**: 8 个模块中仅 3 个完整接入 DataBridge 实时数据管道。ABModelComparisonWidget 和 EventReplayWidget 是完全孤立的"演示级"组件，仅通过手动 API 交互，未与系统数据流打通。WaveformScrollWidget 因信号名拼写错误 (`frame_ready` vs `frame_result_ready`) 导致数据链路中断，且被 `try/except` 静默吞没。

**修复优先级**: Fix 1 (波形信号名) > Fix 2+3 (A/B对比+事件回放集成) > Fix 4 (间接耦合) > Fix 5+6 (硬编码+异常吞没)
