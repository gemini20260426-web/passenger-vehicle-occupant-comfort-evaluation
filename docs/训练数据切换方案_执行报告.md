## 三、模块详细说明

### 3.1 `label_quality_validator.py` — 标签质量预检

**8 项质量检查**：

| # | 检查项 | 严重级别 |
|---|--------|---------|
| 1 | 必要列缺失 (event, t_start, t_end) | ERROR |
| 2 | 空值检查 | ERROR |
| 3 | 时间范围对齐 (t_start/t_end 是否超出数据范围) | WARNING |
| 4 | 事件名映射完整性 | ERROR |
| 5 | 事件分布统计 | INFO |
| 6 | 罕见事件检测 (< 3 样本 → 需合成补充) | WARNING |
| 7 | 事件持续时间异常 (duration <= 0) | WARNING |
| 8 | 重叠事件窗口检测 (> 30% 重叠) | WARNING |

**使用方式**：
```bash
# 检查所有配对
python -m core.core.analysis.label_quality_validator --data_dir data_output

# 检查单个文件
python -m core.core.analysis.label_quality_validator \
    --event_csv data_output/expert_evaluation_20260606_095159/event_analysis.csv \
    --parsed_csv data_output/parsed_data_20260606_094258.csv
```

### 3.2 `batch_training_data_generator.py` — 批量训练数据生成器

**核心功能**：
- 自动扫描 `data_output` 目录，基于时间戳匹配 parsed_data ↔ expert_evaluation
- 批量调用 `FastTrainingDataGenerator` 生成 55 维特征
- 合并所有配对数据，输出统一 `.npz` 文件
- 输出全局类分布统计

**数据配对逻辑**：
```
parsed_data_20260606_094258.csv → expert_evaluation_20260606_095159/
parsed_data_20260606_100404.csv → expert_evaluation_20260606_101323/
parsed_data_20260606_104537.csv → expert_evaluation_20260606_110127/
parsed_data_20260606_112655.csv → 无匹配 (跳过)
parsed_data_20260606_175825.csv → 无匹配 (CAN 数据, 跳过)
```

**使用方式**：
```bash
python -m core.core.analysis.batch_training_data_generator \
    --data_dir data_output \
    --output training_data_real.npz \
    --window_size 500 \
    --step_size 250
```

### 3.3 `adaptive_balancer.py` — 阶梯式类别均衡

**4 级自适应策略**：

| 级别 | 条件 | 策略 | 示例 |
|------|------|------|------|
| Level 0 | 样本 < 3 | 从合成数据补充 | emergency_braking: 2 → 合成补充 8 |
| Level 1 | 样本 < 10 | SMOTE 过采样 | aggressive_accel: 5 → SMOTE → 15 |
| Level 2 | 样本 > 90分位 | 欠采样 | constant_speed: 500 → 欠采样 → 200 |
| Level 3 | 其他 | 保持原样 | normal_accel: 120 → 保持不变 |

**使用方式**：
```bash
python -m core.core.analysis.adaptive_balancer \
    --data training_data_real.npz \
    --min_samples 10 \
    --output training_data_balanced.npz
```

### 3.4 `progressive_trainer.py` — 渐进式混合训练

**5 阶段渐进式**：

| 阶段 | 合成比例 | 真实比例 | 说明 |
|------|---------|---------|------|
| Phase 0 | 100% | 0% | 纯合成数据基准模型 |
| Phase 1 | 90% | 10% | 真实数据注入 10% |
| Phase 2 | 50% | 50% | 合成/真实 50/50 混合 |
| Phase 3 | 10% | 90% | 真实数据主导 90% |
| Phase 4 | 0% | 100% | 纯真实数据模型 |

### 3.5 `sensor_fault_augmenter.py` — 传感器故障数据增强

**4 种故障模式**：

| 故障类型 | 操作 | 严重度 |
|---------|------|--------|
| `signal_stuck` | 信号卡滞 — 重复同一值 | 中 |
| `signal_dropout` | 信号丢包 — 10-30% 随机置零 | 高 |
| `signal_saturation` | 饱和削波 — 截断到限幅值 | 中 |
| `signal_noise_burst` | 噪声爆发 — 加高斯噪声 | 低 |

**使用方式**：
```bash
python -m core.core.analysis.sensor_fault_augmenter \
    --data training_data_real.npz \
    --n_samples 50 \
    --output training_data_with_fault.npz
```

### 3.6 `train_lgbm_model.py` — 新增参数

| 参数 | 说明 |
|------|------|
| `--event_csv` | 事件标注 CSV 路径 (event_analysis.csv) |
| `--data_dir` | data_output 目录 (批量模式，自动配对) |
| `--progressive` | 启用渐进式训练 (合成 + 真实 5 阶段) |

## 四、完整执行流程

### 4.1 快速开始 (一键渐进式训练)

```bash
python -m core.core.analysis.train_lgbm_model \
    --data_dir data_output \
    --progressive \
    --max_samples 50000
```

此命令会自动执行：
1. 标签质量预检
2. 批量训练数据生成
3. 传感器故障数据增强
4. 阶梯式类别均衡
5. 5 阶段渐进式训练
6. 模型验证与保存

### 4.2 分步执行 (调试模式)

```bash
# Step 0: 标签质量预检
python -m core.core.analysis.label_quality_validator --data_dir data_output

# Step 1: 批量生成真实训练数据
python -m core.core.analysis.batch_training_data_generator \
    --data_dir data_output \
    --output training_data_real.npz

# Step 2: 传感器故障数据增强
python -m core.core.analysis.sensor_fault_augmenter \
    --data training_data_real.npz \
    --n_samples 50 \
    --output training_data_with_fault.npz

# Step 3: 阶梯式类别均衡
python -m core.core.analysis.adaptive_balancer \
    --data training_data_with_fault.npz \
    --output training_data_balanced.npz

# Step 4: 单次训练 (真实数据)
python -m core.core.analysis.train_lgbm_model \
    --data training_data_balanced.npz

# Step 5: 渐进式训练 (全流程)
python -m core.core.analysis.train_lgbm_model \
    --data_dir data_output \
    --progressive
```

### 4.3 单文件训练模式

```bash
# 单个 parsed_data CSV + 事件标注
python -m core.core.analysis.train_lgbm_model \
    --csv data_output/parsed_data_20260606_094258.csv \
    --event_csv data_output/expert_evaluation_20260606_095159/event_analysis.csv \
    --max_samples 50000
```

## 六、风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 4 个 parsed_data 数据量不足 | 训练样本过少 | 降低 window_size/step_size 增加密度 |
| 事件标注与 IMU 时间轴不对齐 | 标签错误 | 标签质量预检 (Step 0) 自动检测 |
| 某些事件类型 < 3 样本 | SMOTE 失败 | 阶梯式均衡 → 合成数据补充 |
| sensor_fault 类无真实样本 | 缺失 1 类 | 传感器故障数据增强 (4 种模式) |
| 真实数据准确率低于合成 | 模型退化 | 渐进式训练追踪，可选最佳阶段 |
