# 循环导入问题解决方案

## 问题分析

通过查看代码，发现存在以下循环导入：

1. `core/data_processing/data_source_manager.py` 导入了 `modules.ui.cnap_visualization.CNAPVisualizationWidget`
2. `modules/ui/monitoring_dashboard.py` 导入了 `core.data_processing.data_source_manager.DataSourceManager`

这种循环依赖导致在导入模块时出现 `ImportError`，无法正常初始化相关类。

## 解决方案

有三种可行的解决方案：

### 方案1：使用延迟导入（推荐）

在 `data_source_manager.py` 中，将 `CNAPVisualizationWidget` 的导入移到实际使用它的函数内部：

```python
# 修改前
from modules.ui.cnap_visualization import CNAPVisualizationWidget

# 修改后
def some_function():
    from modules.ui.cnap_visualization import CNAPVisualizationWidget
    # 使用 CNAPVisualizationWidget 的代码
```

### 方案2：创建中间模块

创建一个 `common` 模块来存储共享的类型和接口：

1. 创建 `core/common/types.py` 存储共用的枚举和类型定义
2. 创建 `core/common/interfaces.py` 定义接口类

### 方案3：重构代码结构

将 `CNAPVisualizationWidget` 移到一个不依赖 `data_source_manager` 的模块中，或者将 `DataSourceManager` 的部分功能拆分到独立模块。

## 实施建议

推荐采用**方案1：使用延迟导入**，因为它对现有代码结构改动最小，且能快速解决问题。

下面是具体的修改步骤：

1. 修改 `core/data_processing/data_source_manager.py` 文件，移除顶部的 `CNAPVisualizationWidget` 导入
2. 在使用 `CNAPVisualizationWidget` 的函数内部添加导入语句
3. 确保所有使用 `CNAPVisualizationWidget` 的地方都使用延迟导入

完成这些修改后，循环导入问题将得到解决，测试脚本将能够正常运行。
